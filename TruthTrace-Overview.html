1 · What is TruthTrace?

TruthTrace is a production web application that takes any AI-generated document, automatically extracts every factual claim it contains, verifies each claim against live sources (Wikipedia + DuckDuckGo), and returns a colour-coded audit report with a 0–100 Trust Score. Results stream to the browser in real time — you see each claim verdict appear one-by-one as the pipeline processes it.


Problem it solvesHow
LLMs confidently hallucinate facts, dates, names, and citationsEvery claim is checked against real web sources, not just the model's memory
Hallucinations are hard to spot manually in long documentsFull document is annotated inline with colour-coded highlights
Verification takes expertise and timeAutomated pipeline completes in ~30–60s per document

4 · How Data Flows

① User fills in API key, loads/pastes a document, clicks "Run Audit"
↳ page.tsx sends POST /api/audit with { text, apiKey, model }

② Backend validates the request (key format, non-empty text)
↳ app/api/audit/route.ts opens a ReadableStream (SSE)

③ Groq LLM call #1 — Claim Extraction
↳ lib/groq.ts → extractClaims() → returns JSON array of { claim, span, type }
↳ Streams event: { type: "claim_start", total: N }

④ For each claim — Evidence Gathering (batches of 3, parallel)
↳ lib/evidence.ts → fetchWikipedia(claim) → Wikipedia REST API summary
↳ lib/evidence.ts → fetchDuckDuckGo(claim) → DuckDuckGo Instant Answer API

⑤ Groq LLM call #2 — Verification (one per claim)
↳ lib/groq.ts → verifyClaim(claim, evidence) → { verdict, confidence, reasoning, quote }
↳ Streams event: { type: "claim_result", result: ClaimResult }

⑥ page.tsx receives SSE events → updates state live
↳ Claim cards appear one-by-one as each result streams in

⑦ On "complete" event — full results rendered
↳ TrustScoreRing + MetricCards + VerdictBar + AnnotatedDoc + ClaimCards

5 · Verdicts & Trust Score

VerdictMeaningExample
✓ Verified Evidence directly confirms the claim "Eiffel Tower completed in 1889" — confirmed by Wikipedia
⚠ Unverified Evidence is absent or inconclusive Obscure details with no Wikipedia/web coverage
✗ Hallucinated Contradicts evidence, or invented (fake names/papers/stats) "Dr. Harold Venkatesh of MIT" — fabricated person & paper

Trust Score formula:

Trust Score = (verified_count / total_claims) × avg_confidence_of_verified_claims

Example: 4 verified out of 12 claims, avg confidence 88% → Score = (4/12) × 88 = 29/100
All verified at 90% → Score = 90/100  |  All hallucinated → Score = 0/100

6 · Key Design Decisions

API Key Never Stored

The Groq key is typed in the UI, sent per-request directly to Groq, and never logged or saved on the server. No .env file needed.

SSE Streaming

Results stream to the browser as each claim finishes. The page updates live — you don't wait 60s for everything at once.

Client-side Charts Only

Recharts can't run on the server (no DOM). Loaded with dynamic(..., {ssr:false}) to prevent hydration errors.

100% Free Infrastructure

Groq free tier, Wikipedia free API, DuckDuckGo free API, Vercel free hobby tier. Zero cost to run.
